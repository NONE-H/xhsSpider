我是一个有十年开经验的老程序猿，在公司待过很多实习生、应届生，也兼职在当培训老师带学生，以下内容分享给准备学习 Vue 的童鞋。1、如果 JavaScript 还比较不熟练的话，先花两周的时间把 JavaScript 再提升一下，建议直接看 《JavaScript高级程序设计》；2、看 Vue 的官方文档，Vue的官方文档对我们中国人很友好，比 React 的文档要更友好一些，基本上接近大白话了。直接新建一个 HTML，然后引入 Vue，开始学习基础用法。这里强调一下：一开始学习的时候不要使用 vue-cli 创建，尤其是在你还不熟悉基于 Node.js 的构建工具时，不要给自己增加复杂度，直接在 script 标签中引入就好了。3、学习基础语法的时候重点把双向绑定、data、props、条件（v-if、v-else）、循环（v-for）、事件处理、计算属性、watch先学一遍，学完这些之后，对 Vue 算是有一些了解了，能写出最基本的代码了。4、有了以上的基础之后，就可以开始学习组件、插槽等内容，这里主要是要了解一下组件化的思想。5、从我带学生的经验来看，学完上面的内容，应该要两周左右的时间，接着就可以使用 vue-cli 去学习了，这个过程中需要了解一下 webpack，至少知道 loader、plugin 是什么，对 .vue 文件是编译成 js 文件要有最基本的了解。6、开始做小项目，我一般会建议先做个 todo-list，做这个小项目基本上可以把学到的所有基础知识用上去。7、学习一下 axios 是怎么用的，公司里肯定会对 axios 进行一层封装，不过不难。8、这时候可以进行业务开发了，写一些基础的业务，简单的组件应该不成问题了。写三个月左右，可以稍微熟练一些了。9、你大概率情况下会用开源的组件库，比如 Element UI、IView、Ant Design Vue，你有一些经验了，应该去尝试看看组件库的源码，特别是当你遇到问题的时候，带着问题去看组件库的源码，你应该去学习一下这个组件是怎么实现的，对组件源码熟悉了，对你后续遇到问题的时候会更有思路。学习高手是怎么封装组件的，对你的业务开发有指导作用。#前端 #web前端学习 #自学前端
一、v-show指令和v-if指令的区别是什么？v-show 仅仅控制元素的显示方式，将 display 属性在 block 和 none 来回切换；而v-if会控制这个 DOM 节点的存在与否。当我们需要经常切换某个元素的显示/隐藏时，使用v-show会更加节省性能上的开销；当只需要一次显示或隐藏时，使用v-if更加合理。二、Vue template 到 render 的过程（1）调用parse方法将template转化为ast（抽象语法树， abstract syntax tree）（2）对静态节点做优化。如果为静态节点，他们生成的DOM永远不会改变，这对运行时模板更新起到了极大的优化作用。（3）生成渲染函数. 渲染的返回值是VNode，VNode是Vue的虚拟DOM节点，里面有（标签名，子节点，文本等等）三、Vue模板渲染的原理是什么？vue中的模板template无法被浏览器解析并渲染，因为这不属于浏览器的标准，不是正确的HTML语法，所有需要将template转化成一个JavaScript函数，这样浏览器就可以执行这一个函数并渲染出对应的HTML元素，就可以让视图跑起来了，这一个转化的过程，就成为模板编译。模板编译又分三个阶段，解析parse，优化optimize，生成generate，最终生成可执行函数render。四、Vue事件绑定原理是什么？每一个Vue实例都是一个Event Bus，当子组件被创建的时候，父组件将事件传递给子组件，子组件初始化的时候是有$on方法将事件注册到内部，在需要的时候使用$emit触发函数，而对于原生native事件，使用addEventListener绑定到真实的DOM元素上。五、Vue为何需要Virtual DOM（虚拟DOM）？（1）具备跨平台的优势（2）操作 DOM 慢，js运行效率高。我们可以将DOM对比操作放在JS层，提高效率（3）提升渲染性能六、Vue 中的 axios 是什么？易用、简洁且高效的http库， 支持node端和浏览器端，支持Promise，支持拦截器等高级配置。

1、什么是插槽🤨🙈🙈1.1 插槽（Slot）是Vue提出来的一个概念，正如名字一样，插槽用于决定将所携带的内容，插入到指定的某个位置，从而使模板分块，具有模块化的特质和更大的重用性。1.2 插槽显不显示、怎样显示是由父组件来控制的，而插槽在哪里显示就由子组件来进行控制2、插槽使用🤨🙈🙈2.1 默认插槽  在子组件中写入slot，slot所在的位置就是父组件要显示的内容2.2 具名插槽  ：🌳在子组件中定义了三个slot标签，其中有两个分别添加了name属性header和footer🌳在父组件中使用template并写入对应的slot名字来指定该内容在子组件中现实的位置2.3 作用域插槽  在子组件的slot标签上绑定需要的值<slot :data="user"></slot>在父组件上使用slot-scope=“user”来接收子组件传过来的值 ；👻👻👻最近一直都在整项目 ，前面的vue笔记还没来得及整理，刚开始写项目比较emo的荒；只写了一个移动端和一个pc端的小项目，目前再写一个大项目，估计还得写十几天吧；老师说vuex学完都可以找工作了😂😂#前端   #web前端   #vue   #vuex    #vue3   #学习打卡
